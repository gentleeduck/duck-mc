## Spurious Diffs (NOT engine bugs)

- **75 vs 2 records**: legacy `.velite/duckUi.json` was generated when `docs/duck-ui/components/**/*.mdx` existed. Current source has only `accordion.mdx` + `changelog.mdx` at top level.
- **slug `docs/duck-ui/accordion` vs `docs/duck-ui/components/accordion`**: same transform; different file paths.

## Fixes Shipped

### 1. github-slugger algorithm + Slugger (`dmc-parser/src/slugger.rs`)

`github_slugify(text)` and `Slugger` (per-document dedupe). Strip non-alphanumeric/`_`/`-`; whitespace runs → single `-`; collapse runs; trim; dedupe with `-N` suffix.

### 2. `Heading.id: Option<String>` (`dmc-parser/src/ast/node.rs`)

Pre-computed by transform; emitters read via `Heading::slug()` (falls back to fresh slugify when `None`).

### 3. `AssignHeadingIds` transform (`dmc-transform/src/builtin/assign_heading_ids.rs`)

Walks document; assigns `id` via document-scoped `Slugger`. Wired into default pipeline FIRST.

### 4. `AutolinkHeadings` switched to prepend mode (`dmc-transform/src/builtin/autolink_headings.rs`)

Configurable `aria_label`/`class_name`/`icon_class_name`. Anchor as `JsxElement` carrying `aria-label` + `className`. Prepended to heading children. Idempotent.

### 5. MDX body shape refactor (`dmc-codegen/src/mdx.rs`)

- `MdxBodyEmitter` tracks `used_intrinsic` + `used_components`
- Output uses single-letter aliases throughout: `Fragment→n, jsx→e, jsxs→i, _components→c, props→a`
- `jsx_tag_ref` returns `c.tag` for intrinsics, bare `CompName` for capitalized components
- Defaults map: `const c = { tag: "tag", ..., ...a.components }` — only includes referenced tags
- Component destructure + `_missingMdxReference` checks emitted only when components referenced
- `_missingMdxReference` definition skipped entirely when zero capitalized components used
- `e()` for 0/1 child, `i()` for multi-child (matches `@mdx-js/mdx`)
- Adjacent text-node parts merged into one literal
- `js_string` picks single quotes when content has `"` and no `'`

### 6. Inline `<code>` carries `__rawString__` (`dmc-codegen/src/mdx.rs`)

`Node::InlineCode` emits `e(c.code, { __rawString__: "raw", children: "raw" })`.

### 7. Block-spacing whitespace text nodes (`dmc-codegen/src/mdx.rs`)

`pad_block_container` + `intersperse_block_newlines` + `has_block_jsx_callee` work together to emit the literal `"\n"` text children velite places between block-level siblings inside `<ul>`, `<ol>`, `<li>` (and at root). Only emitted when at least one neighbor is a block-level JSX call.

### 8. Lexer fix: ordered-list marker only at column 1 (`dmc-lexer/src/utils.rs`)

`'0'..='9' if peek() == '.' && self.column == 1`. So `0.4.3` inside heading/paragraph stays as single text token.

### 9. Paragraph soft-break compliance (`dmc-parser/src/block.rs`)

`parse_paragraph` keeps soft-breaks inside the paragraph (CommonMark). Closes only on blank line, block-starter token, or setext underline.

### 10. Aggressive minifier (`dmc-core/src/engine/utils.rs`)

`minify_js` rewritten to drop whitespace adjacent to punctuation, keep only between two ident-like chars. `mdx_minify` default flipped to `true` in `CompileConfig` and `dmc-napi`.

### 11. `rename_idents` utility (`dmc-core/src/engine/utils.rs`)

State-aware identifier substitution that respects string + comment boundaries. Available for downstream use; unused in current pipeline since codegen emits short names directly.

### 12. wordCount split (`dmc-core/src/engine/accumulator.rs`)

- `wordCount`: count from markdown body (post-frontmatter), strip fenced code blocks AND ATX heading markers
- `readingTime`: count from plain-text extraction, divide by 200, round half-up

This split mirrors what velite does (different sources for the two fields). Tested: 906/908 wordCount, exact 3/3 readingTime.

## Remaining Gap: 2-word wordCount drift

Body is byte-exact. wordCount differs by 2 words (906 vs 908) — likely a tokenization quirk where velite's `reading-time` package counts an apostrophe-bridged token (`primitive's`) once where our whitespace splitter agrees. Within `0.22%`; cosmetic.

## What Closed the Last 13 Bytes

Three pieces landed together:

1. **Loose-list multi-line paragraphs** — `parse_list`'s loose-list continuation branch now eats consecutive indented lines into one Paragraph with literal `\n` text nodes between them, instead of emitting a separate Paragraph per source line.

2. **`ensure_loose_item` only promotes leading inlines** — when a list item already has a trailing block child (e.g. nested `<ul>`), the leading text run gets wrapped in `<p>` but the block stays untouched. Previously the helper packed both into one paragraph and swallowed the nested list.

3. **Default-export factory** — codegen emits velite's `MDXContent` factory verbatim: `return { default: function(n={}){ const{wrapper:i}=n.components||{}; return i?e(i,{...n,children:e(_createMdxContent,{...n})}):_createMdxContent(n) } };`

4. **Minifier drops `;` before `}`** — final-statement semicolons are unnecessary; matching velite's stripping eliminates the last 1-byte diff.
