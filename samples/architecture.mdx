# duck-md compiler architecture

End-to-end view of every layer, the data that flows between them, and the diagnostic engine that watches the whole pipeline. Every box is a real Rust type or fn in the workspace.

## Lexer internals

```mermaid
flowchart TD
  Bytes["&'src str source"] --> Cursor

  subgraph LEXER["Lexer&lt;'eng, 'src&gt;"]
    direction TB
    Cursor["Cursor state:<br/>start, current,<br/>line, column,<br/>frontmatter_reserved"]
    LexTokens["lex_tokens(c) — dispatch"]
  end

  Cursor --> LexTokens

  LexTokens --> Modules

  subgraph MODULES["Per-construct lex modules"]
    direction TB
    M1["fontmatter.rs<br/>--- yaml ---"]
    M2["code.rs<br/>``` fences + inline `…`"]
    M3["jsx.rs<br/>&lt;Tag attr={expr}&gt;"]
    M4["lists.rs<br/>- / 1. / [ ] / [x]"]
    M5["statements.rs<br/>import / export"]
    M6["typography.rs<br/>**bold** / _italic_ / ~~strike~~"]
    M7["whitespaces.rs<br/>spaces, newlines, breaks"]
  end

  Modules --> Emit

  Emit["emit(TokenKind)<br/>→ Token{kind, span, raw}<br/>(skips trivia unless line-leading 4-space indent)"]

  Emit --> Token
  Token["Token&lt;'src&gt; {<br/>kind: TokenKind,<br/>span: Span,<br/>raw: &amp;'src str  ← zero-copy<br/>}"]

  Token --> Stream["Vec&lt;Token&lt;'src&gt;&gt;"]

  Cursor -. on error .-> EmitDiag
  EmitDiag["emit_diagnostic(Diagnostic)<br/>→ E001..E010 errors,<br/>   W001 warnings"]
  EmitDiag --> Engine["DiagnosticEngine&lt;Code&gt;"]
```

**Key properties:**
- **Zero-copy**: `Token.raw` is `&'src str` — points into source buffer, no String allocation per token.
- **memchr SIMD**: `skip_until_byte`, `skip_while_byte` use vectorized scanners.
- **Recovery**: errors emit + advance, never abort.
- **State**: `frontmatter_reserved` flips on first `---…---` block so subsequent `---` lines become thematic breaks unambiguously.

## Parser internals

```mermaid
flowchart TD
  Tokens["Vec&lt;Token&gt; from lexer"] --> ParserStruct

  subgraph PARSER["Parser&lt;'eng, 'tokens&gt;"]
    direction TB
    ParserStruct["Parser {<br/>tokens, pos,<br/>meta: Arc&lt;SourceMeta&gt;,<br/>engine<br/>}"]
    ParseLoop["parse() — top-level dispatch loop"]
  end

  ParserStruct --> ParseLoop
  ParseLoop --> Block

  Block["parse_block() peeks token kind →<br/>routes to specialised parser"]

  Block --> P1["parse_frontmatter<br/>(yaml stays raw)"]
  Block --> P2["parse_heading<br/>(# / setext)"]
  Block --> P3["parse_code_block<br/>(``` fenced / 4-space indent)"]
  Block --> P4["parse_list<br/>(unordered / ordered / task)"]
  Block --> P5["parse_blockquote<br/>(>)"]
  Block --> P6["try_parse_table<br/>(GFM | header | : alignment :)"]
  Block --> P7["parse_jsx<br/>(elements / fragments / self-close)"]
  Block --> P8["parse_jsx_expression<br/>({…})"]
  Block --> P9["consume_import / consume_export"]
  Block --> P10["parse_paragraph<br/>(fallback)"]

  P1 --> Inline
  P2 --> Inline
  P10 --> Inline
  P5 --> Inline

  Inline["collect_inline()<br/>Text / Bold / Italic / Strike /<br/>InlineCode / Link / Image / Autolink /<br/>Bracket / Bang / JsxOpenTag /<br/>ExpressionStart / MarkdownComment"]

  Block --> Doc
  Inline --> Doc

  Doc["Document {<br/>children: Vec&lt;Node&gt;,<br/>span<br/>}"]

  ParserStruct -. on error .-> Diag
  Diag["diag/warn → P001..P014, PW001..PW004"]
  Diag --> Engine["DiagnosticEngine"]
```

**Key properties:**
- **Pure structure**: AST is MDX-only — no derived fields. `Heading.id` is computed lazily via `Heading::slug()`. `Frontmatter.data` parsed at compile boundary, not by parser.
- **Real spans**: every node carries the originating token's span (not `default_span()`).
- **Force-advance**: a no-progress iteration advances anyway, so malformed tokens can't wedge the parser.
- **Recovery**: unterminated JSX synthesises a self-close + emits PW004.

## Transform pipeline internals

```mermaid
flowchart TD
  Doc["Document AST<br/>(from parser)"] --> Pipe

  subgraph PIPE["Pipeline"]
    direction TB
    PipelineNew["Pipeline::new() / Pipeline::with_defaults()"]
    Run["pipeline.run(doc, &amp;meta, engine)"]
    Walker["walk_root(&amp;mut Vec&lt;Node&gt;, &amp;mut visitor)"]
  end

  PipelineNew --> Run --> Walker

  Walker --> Actions

  Actions["NodeAction"]
  Actions --> A1["Keep → recurse children"]
  Actions --> A2["KeepSkipChildren → no recurse"]
  Actions --> A3["Replace(Vec&lt;Node&gt;) → splice"]
  Actions --> A4["Remove → drop from parent Vec"]

  Run --> T1
  Run --> T2
  Run --> T3
  Run --> T4
  Run --> T5
  Run --> T6
  Run --> T7
  Run --> T8
  Run --> T9

  T1["AutolinkHeadings<br/>wraps heading children in &lt;a href='#slug'&gt;"]
  T2["BareUrlAutolink<br/>http(s)://… in Text → Link"]
  T3["CodeImport<br/>file=path{ranges} → file contents<br/>diagnostics: T001, T002, TW004"]
  T4["ComponentSource<br/>&lt;ComponentSource path=… /&gt; → CodeBlock<br/>diagnostics: T007, TW002, TW004"]
  T5["ComponentPreview<br/>&lt;ComponentPreview name=… /&gt; → CodeBlock<br/>diagnostics: T003..T006, TW002"]
  T6["CopyLinkedFiles<br/>copy assets, hash-rename, rewrite href/src<br/>diagnostics: TW003, T008"]
  T7["DisableGfm<br/>strikethrough/table/tasklist → plain"]
  T8["Mermaid (cfg)<br/>``` mermaid fence → &lt;MermaidSvg svg=… /&gt;<br/>L1 mem + L2 disk SVG cache<br/>diagnostics: TW001, T009"]
  T9["NpmCommand (cfg)<br/>npm install → &lt;PackageManagerTabs npm yarn pnpm bun /&gt;"]

  T3 -. error queue .-> Pending
  T4 -. error queue .-> Pending
  T5 -. error queue .-> Pending
  T6 -. error queue .-> Pending
  T8 -. error queue .-> Pending
  Pending["Apply.pending: Vec&lt;Diagnostic&lt;Code&gt;&gt;<br/>(visitors can't borrow engine,<br/>so they buffer + drain on finish)"]
  Pending --> Engine["DiagnosticEngine"]
```

**Key properties:**
- **Walker honours all 4 NodeAction variants** — Replace splices `Vec&lt;Node&gt;`, Remove drops the index, Keep recurses, KeepSkipChildren stops descent.
- **`meta` threaded through** — transformers like CodeImport derive `base_dir` from `meta.origin` when File-backed; emit `BaseDirNotFound` warning for Stdin/Inline/Memory.
- **Pure helpers on the public struct** — `CodeImport::parse_file_meta`, `Mermaid::render_mmdc`, etc. State-touching code lives on `Apply`.

## Codegen internals

```mermaid
flowchart TD
  TransformedDoc["transformed Document"] --> Split

  Split{"output target"}
  Split --> Path1
  Split --> Path2

  subgraph PATH1["HtmlEmitter&lt;'a&gt;"]
    direction TB
    HtmlNew["HtmlEmitter::render(doc, &amp;mut engine)"]
    HtmlEmit["emit(node) — variant dispatch"]
    HtmlSpec["JSX special-cases:<br/>&lt;MermaidSvg svg /&gt; → raw inline SVG<br/>&lt;PackageManagerTabs … /&gt; → tab-styled &lt;pre&gt;s"]
    HtmlOut["HTML String<br/>classes prefixed: gentleduck-md-*"]
    HtmlNew --> HtmlEmit --> HtmlSpec --> HtmlOut
  end

  subgraph PATH2["MdxBodyEmitter&lt;'a&gt;"]
    direction TB
    MdxNew["MdxBodyEmitter::render(doc, &amp;mut engine)"]
    HoistIE["hoist Imports / Exports → prelude"]
    EmitJs["emit each node as jsx(…) / jsxs(…)"]
    Wrap["wrap body in:<br/>function _createMdxContent(props) { … }"]
    MdxOut["JS body string<br/>(MDX runtime invokes with<br/>{Fragment, jsx, jsxs})"]
    MdxNew --> HoistIE --> EmitJs --> Wrap --> MdxOut
  end

  Path1 -. JsxExpression dropped .-> GW002["GW002 HtmlExpressionDropped"]
  Path2 -. Table dropped .-> GW001["GW001 MdxTableUnsupported"]
  Path1 -. empty JSX name .-> G001["G001 MalformedJsxTagName"]
  Path2 -. empty JSX name .-> G001
  G001 --> Engine["DiagnosticEngine"]
  GW001 --> Engine
  GW002 --> Engine
```

**Key properties:**
- **Engine plumbed like other layers** — `&mut DiagnosticEngine&lt;Code&gt;` parameter, no buffer-and-drain ceremony.
- **Class prefix** — every emitted CSS class starts with `gentleduck-md-` to avoid collision with user / framework styles.
- **Synthetic JSX** — transforms produce `&lt;MermaidSvg /&gt;` and `&lt;PackageManagerTabs /&gt;`; HTML emitter unwraps them server-side, MDX leaves them for the user's React component map.

## Cross-cutting: DiagnosticEngine

```mermaid
flowchart LR
  subgraph DUCK["duck-diagnostic crate"]
    direction TB
    Engine["DiagnosticEngine&lt;C: DiagnosticCode&gt;"]
    Diagnostic["Diagnostic&lt;C&gt; {<br/>code, severity, message,<br/>labels, notes, help, suggestions<br/>}"]
    Span["Span {<br/>file: Arc&lt;str&gt;,<br/>line, column, length<br/>}"]
    Severity["Severity {<br/>Bug | Error | Warning | Note | Help<br/>}"]
  end

  subgraph DUCKMD["duck-md-diagnostic crate"]
    direction TB
    Code["enum Code (gated by feature flags)"]
    LexerCodes["lexer feature → E001..E010, W001"]
    ParserCodes["parser feature → P001..P014, PW001..PW004"]
    TransformCodes["transform feature → T001..T009, TW001..TW004"]
    CodegenCodes["codegen feature → G001, GW001..GW002"]
    SourceMeta2["metadata::SourceMeta + Origin"]
  end

  DUCKMD -- "implements DiagnosticCode for Code" --> DUCK
  Engine -- emits → --> Diagnostic
  Diagnostic -- carries → --> Span
  Diagnostic -- carries → --> Severity
  Diagnostic -- carries → --> Code

  Lex["Lexer"] --> Engine
  Parse["Parser"] --> Engine
  Trans["Transform"] --> Engine
  Codegen["Codegen"] --> Engine
  Engine -- format_all_plain → --> Stderr["human-readable output"]
  Engine -- iterate → --> Json["JSON / structured output"]
```

**Key properties:**
- **Single shared type across all layers** — `DiagnosticEngine&lt;Code&gt;` is the mailbox; layers borrow it mutably and emit.
- **Feature-gated codes** — `duck-md-lexer` only sees lexer codes, `duck-md-codegen` only sees codegen codes. Top-level `duck-md-core` enables every feature → full enum.
- **Per-token Span sharing via Arc&lt;str&gt;** — one allocation for the file path, every span clones the Arc cheaply.

## Per-runner CLI bins

| Crate | Bin | Default sample | Purpose |
|---|---|---|---|
| `duck-md-lexer` | `lexer` | `samples/errors/*.mdx` (loops) | dump tokens (table or JSON) |
| `duck-md-parser` | `parse` | `samples/index.mdx` | dump AST tree (or JSON) |
| `duck-md-transform` | `transform` | `samples/index.mdx` | before / after AST diff |
| `duck-md-codegen` | `codegen` | `samples/index.mdx` | AST + HTML / MDX body |

All four read from the shared `samples/` dir, accept `-` for stdin, support `--json`, `--quiet`, etc. Drop-in interchangeable.

## Summary in one sentence

`bytes` → **Lexer** (zero-copy tokens) → **Parser** (pure-structure AST) → **Transform** (visitor pipeline mutating in place / splicing) → **Codegen** (HTML + JS body for MDX runtime) → **Core** wraps everything into `CompileOutput`, with a single `DiagnosticEngine` carrying typed `Code`s from every layer.
