## Philosophy

Accordions solve the progressive disclosure problem - show the outline,
hide the details until requested. We use a compound component pattern
(`Accordion` -> `AccordionItem` -> `AccordionTrigger` + `AccordionContent`)
because each piece needs independent control while sharing open/close
state.

## API

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `type` | `"single" \| "multiple"` | `"single"` | Whether one or many items can be open. |
| `collapsible` | `boolean` | `false` | Allow closing the open item. |
| `defaultValue` | `string \| string[]` | `-` | Initial open item value(s). |
| `dir` | `"ltr" \| "rtl"` | `"ltr"` | Reading direction. |
